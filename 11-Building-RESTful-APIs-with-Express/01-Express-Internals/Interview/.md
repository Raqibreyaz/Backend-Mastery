## 01. Express-Internals (10 questions)

1. How does Express's request-response cycle work internally, from the moment a request hits the server to when the response is sent?
2. What is the role of `app._events` in Express, and how does it relate to the internal event emitter mechanism?
3. Explain how Express's `router` module internally manages route stacks and dispatches requests.
4. What happens internally when you call `app.use()` without a path argument? How does Express handle this differently than `app.use('/path')`?
5. How does Express's `Application` constructor set up the internal middleware stack and router configuration?
6. What is the difference between `app.handle()` and `app.dispatch()` in Express's internal flow?
7. Explain how Express parses and normalizes middleware functions internally before adding them to the stack.
8. How does Express internally track and manage the "done" flag to prevent middleware from being called multiple times?
9. What internal mechanisms does Express use to handle async errors versus regular errors in middleware?
10. How does Express's `layer.process_request()` method work internally when processing a route layer?
