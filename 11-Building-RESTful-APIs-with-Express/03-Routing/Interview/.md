## 03. Routing (10 questions)

1. How does Express's routing mechanism match requests to routes, and what is the order of evaluation?
2. Explain the difference between `app.get()`, `app.post()`, `app.all()`, and `app.route()`. When would you use each?
3. How do route parameters work in Express? Compare `/:param`, `/:param?`, `/:param*`, and `/:<param>` patterns.
4. What is the difference between using `app.use('/path', router)` and defining routes directly with `app.get('/path')`?
5. How do you implement route chaining and what are the benefits of using `router.route()` over multiple `app.method()` calls?
6. Explain how Express handles regex patterns in routes. What are the performance implications?
7. How do you create a custom routing middleware that can modify request parameters before reaching the final handler?
8. What happens when multiple routes match the same request? How does Express determine which one executes first?
9. How do you implement dynamic route generation (e.g., creating routes based on database configuration)? What are the security considerations?
10. Explain the difference between `next('route')` and `next()` in route handlers. When would you use each?
