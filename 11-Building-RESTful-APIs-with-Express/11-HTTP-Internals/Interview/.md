## 11. HTTP-Internals (10 questions)

1. Explain the complete HTTP request-response cycle in Express, from TCP connection to response sent, including all layers.
2. How does Express handle HTTP methods (GET, POST, PUT, DELETE, etc.), and what happens internally when an unsupported method is used?
3. What is the difference between HTTP/1.1, HTTP/2, and HTTP/3, and how does Express support each (including implications for middleware)?
4. How do HTTP headers work in Express, and what are the differences between `req.headers`, `req.get()`, and `res.set()`?
5. Explain how Express handles HTTP status codes, including the difference between `res.status()`, `res.statusCode`, and automatic status setting.
6. What is request streaming in Express, and how does it differ from buffered requests? When should you use streaming?
7. How does Express handle HTTP keep-alive, and what are the performance implications for long-running applications?
8. Explain how Express processes query parameters and URL parsing, including the difference between `req.query`, `req.params`, and `req.body`.
9. What are common HTTP security headers you should set in Express (CSP, HSTS, X-Frame-Options), and how do you implement them?
10. How do you debug HTTP issues in Express applications (network logs, header inspection, status code analysis), and what tools work best?
