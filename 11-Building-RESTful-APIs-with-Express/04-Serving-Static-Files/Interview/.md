## 04. Serving-Static-Files (10 questions)

1. How does `express.static()` work internally, and what are its default behaviors for file serving?
2. What are the key configuration options for `express.static()`, and how do they affect performance and security?
3. Explain the difference between serving static files with `express.static()` versus using `res.sendFile()` or `res.download()`.
4. How does Express handle caching headers for static files, and how can you customize this behavior?
5. What security considerations should you keep in mind when serving static files, and how do you prevent directory traversal attacks?
6. How do you serve static files from multiple directories, and what are the performance implications of each approach?
7. Explain how Express handles file extensions and MIME types when serving static files. How can you override default behavior?
8. What is the difference between serving static files at the root level versus using a specific path prefix (e.g., `express.static('public', {fallthrough: false})`)?
9. How do you implement custom error handling for missing static files, and what HTTP statuses should you return?
10. What are best practices for optimizing static file delivery in production (compression, CDN integration, caching strategies)?
