## 08. REST-API (10 questions)

1. What are the core principles of REST, and how does Express support implementing them effectively?
2. How do you design proper HTTP status codes for different REST API scenarios (success, errors, validation, authentication)?
3. Explain the difference between PUT, PATCH, POST, and DELETE in REST, and how to implement them correctly in Express.
4. How do you implement versioning in REST APIs using Express, and what are the trade-offs of each approach (URL, header, query param)?
5. What are best practices for structuring REST API endpoints in Express (folder structure, naming conventions, separation of concerns)?
6. How do you implement pagination, filtering, and sorting in REST APIs using Express query parameters?
7. Explain how to handle hypermedia (HATEOAS) in REST APIs with Express, and when it's appropriate to use.
8. How do you implement rate limiting for REST APIs in Express, and what are the considerations for different user types?
9. What are common REST API security vulnerabilities, and how do you protect against them in Express (authentication, input validation, HTTPS)?
10. How do you test REST APIs built with Express, and what tools/frameworks work best for unit testing, integration testing, and end-to-end testing?
